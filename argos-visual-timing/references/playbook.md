# Argos 视觉截图时机 playbook

## 适用场景

这个 playbook 适用于 Playwright 视觉测试、Argos baseline 复盘、以及“页面看起来没坏，但截图一直在漂”的情况。

## 快速判断

### 1. 先看 build diff

- changed 多，先判断是不是时机漂移而不是视觉重构
- unchanged 多，说明当前大部分页面态已经稳，优先修少数例外

### 2. 按截图名分组

- 入口页：优先早截
- 概览页：通常需要轻微 settle
- 动态列表页：等列表/卡片锚点
- 商城/奖励/预览页：常见“内容晚到”，不要只靠 spinner 消失

### 3. 找锚点

优先用这些元素作为截图前置条件：

- 主按钮可见
- 列表容器可见
- 关键卡片可见
- tab 栏可见
- 页面标题可见

如果这些锚点已经稳定，通常不需要继续加固定等待。

## 这次实践里的范式

- `hall-home-login-entry`
  - 早截
  - 跳过 spinner 等待
  - settle 设为 0
- `hall-home-overview-guest`
  - 比默认稍晚一点
  - 目的是让概览内容完全展开
- `game-entry-shell`
  - 用房间组文本和赔率条作为锚点
- `daily-bonus-entry`
  - 用奖励说明文本稳定后再截
- `mall-preview-shell`
  - 先进入设置页，再等商城预览卡片和 tabs 可见

## 不推荐的做法

- 把所有页面统一加长等待
- 只看 spinner，不看内容是否已经稳定
- 同一个页面同时保留多个只差几十毫秒的截图
- 在没有锚点时先加大 timeout，而不是先找页面状态信号

## 处理原则

1. 优先减少重复态。
2. 优先用页面状态而不是纯时间。
3. 只让少数例外页拥有特殊等待。
4. 如果截图还在漂，先收紧锚点，再考虑调毫秒数。
